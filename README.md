# Cybersecurity Analysis Series

**Prepared by:** Alister A. Rodrigues — Security Analyst, Obsidian SecureTech Solutions

## Overview

This repository is a collection of focused security analyses, reference architectures, and threat modeling exercises spanning core cybersecurity domains — authentication, identity management, PKI, data threat modeling, and governance frameworks.

Each piece in this series represents targeted analytical work on a specific security problem, grounded in real-world standards and industry best practices. Taken together, they form the technical foundation that informed the larger, full-scale consulting engagement documented in the **[OCS Cybersecurity Strategy](https://github.com/alisterrodrigues/ocs-cybersecurity-strategy)** — a comprehensive 105-page security program developed for a complex, multi-market client undergoing digital transformation.

If you want to see how these building blocks scale into a complete enterprise security strategy, start there.

## How This Connects to the OCS Engagement

Each analysis in this series maps directly to a domain covered in the OCS consulting report:

- **Authentication Protocols** → informed the IAM & Authentication Controls section of OCS, including the multi-protocol strategy and authentication protocol analysis table
- **Framework Recommendation** → preceded the multi-framework governance architecture built for OCS (NIST CSF 2.0, ISO 27001, PCI DSS, FedRAMP, CMMC, GDPR)
- **IAM Workflows** → directly preceded the IAM Reference Architecture (Fig 3) in the OCS report
- **Key & Certificate Management** → directly preceded the PKI Architecture (Fig 5 & 6) in the OCS report
- **STRIDE Threat Analysis** → reproduced and expanded in the OCS report appendix for the user profile data management process

The progression is intentional: focused problem → refined methodology → full-scale application.
